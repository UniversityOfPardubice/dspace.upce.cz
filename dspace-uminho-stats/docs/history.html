Changes in Stats 2.1.1

Bug fixes

  • Corrected the class invoked in the script stats-aggregate
  • Added the dependency dspace-stats-api in modules/xmlui/pom.xml

Changes in Stats 2.1

  • Adapted to DSpace 1.5.1 build and deploy architecture
  • Installation process based on SVN Patch mechanism
  • Improvement to the gathering mechanism of DSpace events
    • The add-on no longer depends on DSpace logging system
    • Eliminated the log4j JDBC appender mechanism
    • The events are processed and stored using a event firing mechanism on the user interface
      (based on Mark Wood UsageEvent plug-in)
  • Improvement to the events aggregation mechanism
    • Passed from PostgreSQL environment (pg/plsql) to Java.
    • There is no need to install pl/java anymore.
    • All the additional pl/pgsql functions were ported to Java
    • There is no need to install pl/java anymore.
  • Improvement to the spider detection mechanism
    • Performance improvements of the procedure.
    • Weblog analysis improvment.
    • Sessions analysis.
    • Flagging of IPs to watch in the future
    • User interface to this information (integrated on statistics user interface)
  • New administrator commands (Java API)

Changes in Stats 2.0

  • Improvement of the performance and response time
    • Temporal data aggregations
    • Dimensional aggregations (e.g. Country, Document type, etc.)
  • Improvement of the presentation and navigation
    • Customizable menu navigation Structure (100% XML)
    • Customizable statistics layout (100% XSL stylesheets)
    • Charts
  • New statistics
  • Multilanguage support
  • Better system documentation
    • Renewed Installation Guide
    • New User Guide

Changes in Stats 1.1

  • Detection, processing and distinction of own institution access
  • New 3:4 country flags
  • Dspace 1.4 compatibility
